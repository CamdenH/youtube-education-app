## Milestone Summary

**Key Decisions:**
- Clerk + Supabase + Railway stack confirmed — fastest path to production auth+billing+DB without infra ops
- Clerk Billing over Stripe directly — handles subscription UI + webhooks + user linkage natively
- Global cache (no user_id in key) — MD5 hash of query/video ID; cross-user cache hits save YouTube API quota
- Courses.user_id is TEXT (Clerk ID), not FK to users table — avoids webhook timing race
- Atomic counter via Postgres RPC `increment_generation_count` — not app-level read-then-write (D-07)
- Tier names: free (1/month) / early_access (20/month, $10/month)
- Clerk billing webhook events: subscriptionItem.active / subscriptionItem.ended (NOT subscription.created/deleted)
- HTML route auth gate pattern: inline getAuth() + res.redirect('/') — NOT requireUser (returns 401 JSON, wrong for HTML routes)
- window.__upgradeUrl Option B: hardcoded empty string in pricing.html — no server injection, no XSS
- Marketing pages (landing.html) must not link to /app — all CTAs link to Clerk auth URLs or /pricing
- fetch() preflight before EventSource: native EventSource cannot read HTTP status codes; fetch() catches 429 first

**Issues Resolved:**
- File cache (.cache/) replaced with Supabase cache table — dev-only shortcut eliminated
- localStorage course history replaced with Supabase-backed /api/courses API

**Issues Deferred:**
- Per-user watched state (checkboxes) still in localStorage — Supabase migration deferred to v3.0
- Human verification items (3 VERIFICATION.md human_needed, 4 UAT scenarios) — acknowledged and deferred at milestone close

**Technical Debt Incurred:**
- Watched checkbox state not in Supabase (localStorage only)
- Clerk placeholder values in index.html Clerk CDN script require manual substitution before deploy

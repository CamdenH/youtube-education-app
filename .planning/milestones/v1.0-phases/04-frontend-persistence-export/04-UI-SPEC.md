## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot indicators, tight inline padding |
| sm | 8px | Button padding (vertical), gap between badge items |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, card-to-card gap |
| xl | 32px | Layout section breaks, form margin-bottom |
| 2xl | 48px | Major section breaks (search → course transition) |
| 3xl | 64px | Page-level top/bottom breathing room |

Exceptions:
- Touch targets (checkboxes, accordion toggles) must be minimum 44px hit area even if visually smaller — achieved via padding, not size changes
- Video thumbnail fixed at 120px wide × 68px tall (16:9 ratio, hardcoded)
- Score badge: 28px height, 40px min-width — exception to 4px scale for badge legibility

Source: CSS custom properties already declared in `index.html` `:root` block.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.15 |

Source: CSS custom properties already declared in `index.html` `:root` block — pre-populated, not changed.

Usage rules:
- Display (28px/600): Page title "YouTube Learning Curator" and generated course title only
- Heading (20px/600): Module card titles, section headings (History, Recent Searches)
- Body (16px/400): All prose — course overview, video rationale blurbs, question text, textarea input, pipeline step messages
- Label (14px/400): Score badges, outdated badges, channel names, duration, timestamps, monospace event names in pipeline log, prerequisite badges, "why this video" attribution label

Monospace exception: Pipeline step event names use `font-family: monospace` at 14px/400 — already established in existing stub.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f0f0f` | Page background (`--color-bg`) |
| Secondary (30%) | `#1a1a1a` / `#242424` | Cards, sections, raised surfaces (`--color-surface`, `--color-surface-raised`) |
| Accent (10%) | `#3b82f6` | Reserved list below |
| Success | `#22c55e` | Pipeline completion dot, "watched" checkbox checked state, progress bar on success |
| Destructive | `#ef4444` | Pipeline error dot, error state message text only |

Source: CSS custom properties already declared in `index.html` `:root` block.

Accent (`#3b82f6`) reserved for:
1. Primary CTA button background ("Generate course", "Export course")
2. Progress bar fill during active pipeline run
3. Hyperlinks (video titles linking to YouTube, thumbnail overlay)
4. Focus ring on all interactive elements (2px solid, 2px offset)
5. Active/selected history item left border indicator (3px)
6. Score badge background for the 60-79 tier (mid-quality videos)

Accent is NOT used for: decorative borders, backgrounds, text on non-interactive elements, or module card headers.

Outdated warning color: `#f59e0b` (amber) — used exclusively for the "May be outdated" flag badge background. This is the only amber usage.

Text colors:
- Primary text: `#f0f0f0` (`--color-text`) — all body and heading text
- Muted text: `#a0a0a0` (`--color-text-muted`) — labels, timestamps, placeholder text, channel names

## Interaction States

### Form — Search Screen

**Subject input:**
- Default: border `1px solid #242424`, background `#1a1a1a`, text `#f0f0f0`, placeholder `#a0a0a0`
- Focus: border `1px solid #3b82f6`, focus ring `2px solid #3b82f6` at 2px offset
- Disabled (during generation): opacity 0.5, cursor not-allowed

**Skill level selector:**
- Rendered as a `<select>` element, styled to match input
- Options: Beginner, Intermediate, Advanced, All Levels
- Default selection: All Levels

**Generate button:**
- Default: background `#3b82f6`, text white, border-radius 6px, padding 10px 20px
- Hover: background `#2563eb` (10% darker)
- Active: background `#1d4ed8`
- Disabled (during generation): opacity 0.6, cursor not-allowed, text "Generating..."
- Width: full-width on mobile (375px), auto on wider viewports

### Pipeline Loading State (FRNT-02)

Five labeled steps displayed as a vertical list, each with:
- A 10px dot (colored by state: muted=pending, accent=active/complete, success=done, destructive=error)
- A human-readable label (not the SSE event name)
- No timestamps shown during loading — timestamps appear only in the dev stub, not production UI

Step labels (map from SSE event names):
| SSE Event | Display Label |
|-----------|---------------|
| `query_generated` | Generating search queries |
| `videos_fetched` | Searching YouTube |
| `scored` | Scoring videos |
| `transcripts_fetched` | Fetching transcripts |
| `course_assembled` | Building your course |

Steps animate in sequentially. Active step shows a subtle pulse animation on the dot (CSS keyframe, no JS required). Completed steps show a solid success-color dot. Progress bar sits below the step list; transitions width over 300ms ease.

### Module Cards (FRNT-05)

- Collapsed by default — show module title + description only
- Toggle via `<details>`/`<summary>` HTML element — no JS toggle required
- First module expanded by default (first `<details>` has `open` attribute)
- Expansion shows video cards inside

### Video Cards (FRNT-06)

Thumbnail:
- 120×68px fixed, `object-fit: cover`, border-radius 4px
- Wraps an `<a href="https://youtube.com/watch?v={id}" target="_blank" rel="noopener">`
- On hover: 0.85 opacity overlay

Score badge:
- Pill shape, 28px height, min-width 40px, centered text
- Background color tiers:
  - 80-100: `#22c55e` (success green)
  - 60-79: `#3b82f6` (accent blue)
  - 40-59: `#f59e0b` (amber)
  - 0-39: `#a0a0a0` (muted gray)
- Font: 14px/600 (label size)

Outdated warning badge (CURA-06):
- Text: "May be outdated"
- Background: `#f59e0b`, text: `#0f0f0f` (dark on amber for contrast), border-radius 4px, padding 2px 8px, font 14px/600
- Displayed inline with score badge row

Watched checkbox (PERS-02):
- Native `<input type="checkbox">` visually styled
- Checked state: accent blue border + success green checkmark fill
- Label: "Mark as watched"
- 44px minimum hit target via padding

### Questions Accordion (FRNT-07)

- `<details>`/`<summary>` pattern — "Comprehension questions (3)"
- Collapsed by default on all videos
- Expanded: shows 3 labeled questions (Recall, Conceptual, Application)
- Each question has a `<textarea>` for user answers: background `#1a1a1a`, border `1px solid #242424`, border-radius 4px, min-height 72px, resize: vertical, full width
- Textarea placeholder: "Write your answer here..."
- User answers are NOT persisted (out of scope per REQUIREMENTS.md)

### Reveal Thinking Points Button (FRNT-08)

FRNT-08 is a Phase 4 requirement. The hint generation API (HINT-01 through HINT-03) ships in Phase 5, but the button UI is wired in Phase 4.

- Placement: below the 3 comprehension questions, inside the questions accordion, above the watched checkbox
- Label (idle): "Reveal thinking points"
- Label (loading): "Getting hints..." — button disabled, cursor not-allowed, opacity 0.6
- Label (loaded): "Thinking points" — button switches to a non-interactive label style (no longer a button)
- Hint reveal format: each hint appears inline below its corresponding question as muted text (14px/400, `#a0a0a0`), prefixed with "Hint:" in label weight (14px/600)
- Button style: same secondary style as Export — transparent background, border `1px solid #3b82f6`, text `#3b82f6`, border-radius 6px, padding 6px 14px
- Phase 4 scope: button is rendered and the loading state is wired; actual API call implementation (POST /api/hint) is Phase 5. In Phase 4 the button click handler is a no-op stub that can be replaced in Phase 5.

### History Panel (PERS-01)

- Positioned below search form, above any rendered course
- Heading: "Recent courses" (20px/600)
- Shows last 10 saved courses as a vertical list of clickable rows
- Each row: course title (truncated at 1 line with ellipsis) + skill level badge + relative date (e.g. "3 days ago")
- Active/selected: 3px left border in accent blue, background `#242424`
- Empty state: "No saved courses yet." in muted text, no empty-state card

### Recent Searches (FRNT-03)

- Shows last 5 unique search terms below the subject input field
- Rendered as a row of pill buttons
- Pill: background `#242424`, text `#a0a0a0`, border-radius 100px, padding 4px 12px, font 14px/400
- Hover: background `#333`, text `#f0f0f0`
- Clicking pre-fills the subject input

### Export Button (EXPO-01)

- Secondary button style: background transparent, border `1px solid #3b82f6`, text `#3b82f6`, border-radius 6px, padding 8px 16px
- Positioned in course header row, right-aligned
- Hover: background `rgba(59,130,246,0.1)`
- Label: "Export course"
- On click: triggers markdown file download — no confirmation dialog required (non-destructive)

## Layout

**Max-width:** 640px, centered, horizontal padding 24px on mobile

**Primary visual anchor:** The subject input + Generate course button are the focal center of the search screen. All other elements (recent searches, history panel) are secondary to this pair.

**Page structure (top to bottom):**
1. Page title + subtitle
2. Search form (subject input + skill level selector + generate button)
3. Recent searches pills (FRNT-03)
4. History panel (PERS-01) — hidden when empty
5. Pipeline loading state (FRNT-02) — visible only during generation
6. Course view (FRNT-04 through FRNT-09) — visible after generation completes

**Course header layout:**
- Course title (display size, left)
- Export button (right-aligned, same row on desktop; below title on 375px)
- Overview paragraph
- Prerequisite badge row (horizontal scroll if overflow on mobile)
- Estimated watch time label

**Module card layout:**
- `<details>` block, full width
- Summary row: module title (heading) + video count label (muted, right)
- Expanded content: module description + video cards + connecting question footer

**Video card layout (within module):**
- Horizontal layout: thumbnail left, content right
- Content column: title (body/600), channel + duration row (label/muted), badge row (score + outdated), "Why this video" blurb (label/muted), comprehension questions accordion, reveal thinking points button, watched checkbox
- On 375px: thumbnail stays 120×68px, content column wraps below if needed — min-width 0 on content column to prevent overflow

## Markdown Export Format (EXPO-01)

The downloaded file is named `{course-title-slug}.md`. Content structure:

```
# {Course Title}

**Subject:** {subject} | **Level:** {skill level} | **Watch time:** {estimated time}

## Prerequisites
- {prerequisite 1}
- {prerequisite 2}

## Overview
{overview paragraph}

---

## Module 1: {Module Title}

{Module description}

### {Video Title}
- **Channel:** {channel name}
- **Duration:** {duration}
- **Link:** https://youtube.com/watch?v={videoId}
- **Why:** {rationale blurb}

**Questions:**
1. (Recall) {question text}
2. (Conceptual) {question text}
3. (Application) {question text}

**Module question:** {connecting question}

---
```

Rules:
- No user answers included (out of scope)
- No hint text included (Phase 5 feature)
- No score values included (internal curation signal, not learner-facing)
- Outdated flag preserved as text: "Note: This video may contain outdated information."

## Accessibility Minimums

- All interactive elements keyboard-navigable in logical DOM order
- Focus ring: 2px solid `#3b82f6`, 2px offset — never `outline: none` without a visible alternative
- Thumbnail `<img>` alt text: `"{video title} — YouTube thumbnail"`
- `<details>`/`<summary>` used for all accordions — native keyboard expand/collapse, no ARIA required
- Checkboxes use visible `<label>` association via `for`/`id`
- Color is never the sole differentiator — score tiers use color + numeric value; pipeline steps use color + text label
- Amber (`#f59e0b`) on dark background (`#0f0f0f`) meets 4.5:1 minimum contrast for badge text at 14px/600

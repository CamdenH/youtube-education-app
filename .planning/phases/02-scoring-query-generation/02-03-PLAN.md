**9. scoreVideos(videos, skillLevel) — exported:**
```javascript
async function scoreVideos(videos, skillLevel) {
  const weights = WEIGHTS[skillLevel] || WEIGHTS.intermediate;

  // Extract unique channels for batch call
  const seen = new Set();
  const uniqueChannels = [];
  for (const v of videos) {
    const name = v.snippet.channelTitle;
    if (!seen.has(name)) {
      seen.add(name);
      uniqueChannels.push({
        name,
        subscriberCount: v.statistics?.subscriberCount || null
      });
    }
  }

  // One Claude call for credibility + description quality (SCOR-04, Option A for SCOR-05)
  const { channels: credMap, descriptions: descMap } = await scoreChannelCredibility(uniqueChannels, videos);

  return videos
    .map(video => {
      const likeRatioScore = scoreLikeRatio(video.statistics || {}, weights.likeRatio);
      const durationScore  = scoreDuration(video.contentDetails?.duration || 'PT0S', skillLevel, weights.duration);
      const recencyScore   = scoreRecency(video.snippet.publishedAt, skillLevel, weights.recency);
      const credScore      = Math.min(Number(credMap[video.snippet.channelTitle] || 0), weights.credibility);
      const descScore      = Math.min(Number(descMap[video.id] || 0), weights.description);

      const score = likeRatioScore + durationScore + recencyScore + credScore + descScore;
      return {
        ...video,
        score,
        scoreBreakdown: { likeRatioScore, durationScore, recencyScore, credScore, descScore }
      };
    })
    .sort((a, b) => b.score - a.score);
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YouTube API data → scorer | Video stats arrive as strings from YouTube API — must be coerced before arithmetic |
| Claude batch response → score lookup | Channel names used as keys — mismatch between name in prompt vs name in video object causes silent 0-score |
| scorer output → sse.js | Scored videos passed to Phase 3 pipeline — scoreBreakdown is informational, score is the ranking key |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Tampering | YouTube stat strings | mitigate | Number() coercion before all arithmetic; missing fields default to 0 via `|| 0` guards |
| T-02-03-02 | Information Disclosure | Channel names in Claude prompt | accept | Channel names are public YouTube data; no PII involved |
| T-02-03-03 | Denial of Service | Claude batch call failure | mitigate | callClaude retries 2x with exponential backoff; failure propagates to SSE error handler in server.js |
| T-02-03-04 | Spoofing | Claude returns unexpected JSON shape | mitigate | parseClaudeJSON + shape validation: if channels or descriptions key missing, default to empty object (0-score fallback) |
| T-02-03-05 | Elevation of Privilege | Prompt injection via channel name | accept | Channel names sourced from YouTube API, not user input; personal tool with no adversarial channel names |
</threat_model>

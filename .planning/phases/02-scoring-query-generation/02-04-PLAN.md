New `courseStreamHandler`:

```javascript
async function courseStreamHandler(req, res) {
  const { subject, skill_level: skillLevel } = req.query;

  // Set required SSE headers
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');
  res.setHeader('X-Accel-Buffering', 'no');
  res.flushHeaders();

  const heartbeatInterval = startHeartbeat(res);
  req.on('close', () => clearInterval(heartbeatInterval));

  // Step 1: Generate search queries via Claude
  const queries = await generateQueries(subject, skillLevel);
  sendEvent(res, 'query_generated', {
    step: 1,
    total: 5,
    message: `Generated ${queries.length} search queries`,
    queries,
  });

  // Step 2: Search YouTube for each query and collect unique video IDs
  const searchResults = await Promise.all(queries.map(q => searchVideos(q)));
  const videoIdSet = new Set();
  for (const result of searchResults) {
    for (const item of (result.items || [])) {
      if (item.id?.videoId) videoIdSet.add(item.id.videoId);
    }
  }
  const videoIds = Array.from(videoIdSet);

  // Fetch full stats for all unique videos
  const videos = await fetchVideoStats(videoIds);
  sendEvent(res, 'videos_fetched', {
    step: 2,
    total: 5,
    message: `Fetched ${videos.length} candidate videos`,
  });

  // Step 3: Score and rank videos
  const scoredVideos = await scoreVideos(videos, skillLevel);
  sendEvent(res, 'scored', {
    step: 3,
    total: 5,
    message: `Scored and ranked ${scoredVideos.length} videos`,
    videos: scoredVideos,
  });

  // Steps 4–5: Stubs (Phase 3 will replace these)
  sendEvent(res, 'transcripts_fetched', {
    step: 4,
    total: 5,
    message: 'Transcript fetching coming in Phase 3',
  });

  sendEvent(res, 'course_assembled', {
    step: 5,
    total: 5,
    message: 'Course ready (stub — Phase 3)',
    course: { title: subject, overview: '', modules: [] },
  });

  clearInterval(heartbeatInterval);
  res.end();
}
```

- Do not add error handling inside courseStreamHandler — the try/catch wrapper in server.js handles all errors and maps them to SSE error events.
- Do not add logging beyond what already exists.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/client → GET /api/course-stream | Untrusted query params subject and skill_level cross this boundary — validated in server.js before pipeline |
| sse.js → queries.js / scorer.js | Internal module boundary — inputs already validated at route layer |
| scored video data → SSE response | Video data from YouTube API (public) flows to SSE client — no sensitive data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Tampering (Prompt Injection) | subject query param → Claude prompt | mitigate | validate subject: required, max 200 chars in server.js before courseStreamHandler is called per ASVS V5 |
| T-02-04-02 | Tampering (Input Manipulation) | skill_level query param → WEIGHTS lookup | mitigate | validate skill_level against VALID_LEVELS allowlist in server.js; unknown values return 400 before any processing |
| T-02-04-03 | Denial of Service | concurrent YouTube API calls (Promise.all) | accept | Personal tool; single user; YouTube quota is the natural throttle; .cache/ prevents repeat calls |
| T-02-04-04 | Information Disclosure | scored video list in SSE event | accept | Public YouTube data; no PII; scoreBreakdown reveals algorithm weights (acceptable for personal tool) |
| T-02-04-05 | Denial of Service | malformed subject (e.g., 200-char unicode) | mitigate | length check uses .length (code unit count) — sufficient for personal tool; unicode edge cases not a concern |

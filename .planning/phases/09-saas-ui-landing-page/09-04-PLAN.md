## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → pricing.html | Public static page; Clerk JS loads client-side to detect auth state |
| Clerk.user → upgrade-cta href | Client-side DOM mutation based on auth state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04-01 | Tampering | window.__upgradeUrl CTA swap | mitigate | Guard: `if (Clerk.user && window.__upgradeUrl)` — swap only fires when both conditions true; empty string (Option B default) is falsy so swap is skipped when env var absent |
| T-09-04-02 | Tampering | XSS via window.__upgradeUrl | accept | Option B: value is hardcoded empty string in HTML — no server injection, no user input. If Option A were used, JSON.stringify() would be required. Option B eliminates this vector entirely. |
| T-09-04-03 | Information Disclosure | Clerk publishable key in HTML source | accept | Publishable keys are designed to be public (client-side only, prefixed pk_live_); not a secret |
  | T-09-04-04 | Elevation of Privilege | Pricing page public access | accept | D-12 explicitly requires no auth gate; pricing is a marketing page with no sensitive data |
</threat_model>

## Spacing Scale

Declared values are CSS custom properties copied from index.html `:root` into every new HTML file.

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--space-xs` | 4px | Icon gaps, badge padding, inline tight spacing |
| sm | `--space-sm` | 8px | Compact element spacing, button padding (vertical) |
| md | `--space-md` | 16px | Default element spacing, button padding (horizontal), card inner padding |
| lg | `--space-lg` | 24px | Section padding, page-wrapper horizontal padding |
| xl | `--space-xl` | 32px | Layout gaps, hero bottom margin, section top padding |
| 2xl | `--space-2xl` | 48px | Major section breaks, hero top padding |
| 3xl | `--space-3xl` | 64px | Page-level vertical spacing (footer clearance) |

Exceptions:
- Nav header vertical padding: `--space-lg` top, `--space-xl` bottom (matches existing landing.html pattern)
- Button padding: 8px top/bottom (`--space-sm`) × 16px left/right (`--space-md`) — matches existing `.btn-primary` in landing.html
- Mobile hero headline font-size drops to 22px at ≤480px viewport width

> Source: index.html `:root` lines 10–16; landing.html `.btn-primary` lines 100–111.

## Typography

All sizes map directly to CSS custom properties. No new sizes introduced in this phase.

| Role | CSS Variable | Size | Weight | Line Height | Usage |
|------|-------------|------|--------|-------------|-------|
| Label | `--font-size-label` | 14px | 400 (`--font-weight-regular`) | 1.4 | Nav brand, footer text, muted metadata, pricing card tier label, score badges |

Mobile exception: Display size drops to 22px at ≤480px (matches existing landing.html `@media` rule).

## Accessibility Contract

- All `<a>` elements used as primary buttons carry `href` (not `role="button"`) — links navigate to URLs, not trigger JS.
- Focus state: `outline: 2px solid var(--color-accent); outline-offset: 2px;` on every interactive element.
- Nav landmark: `<header>` wraps the nav. Page content in `<main>`. Footer in `<footer>`.
- Section labels: each `<section>` carries `aria-label` describing its content (matching existing index.html + landing.html pattern).
- Color contrast: `--color-text` (#f0f0f0) on `--color-bg` (#0f0f0f) passes WCAG AA (contrast ratio ~14.7:1). `--color-text-muted` (#a0a0a0) on `--color-bg` (#0f0f0f) passes WCAG AA (contrast ratio ~7.0:1). Accent (#3b82f6) on dark bg passes AA for large text.
- `lang="en"` on `<html>`. `charset="UTF-8"` and viewport meta on every page.
- `<meta name="color-scheme" content="dark">` on every page (matching existing files).

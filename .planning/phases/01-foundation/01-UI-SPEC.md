# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope note:** Phase 1 delivers a minimal `index.html` stub whose sole purpose is to verify the SSE connection end-to-end. It is NOT the final UI (that is Phase 4). The design contract below establishes the token system and stub interaction contract that Phase 4 will build on — defined now so Phase 4 has zero design ambiguity.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | REQUIREMENTS.md: "Framework-based frontend (React/Vue/Svelte) out of scope"; no build step |
| Preset | not applicable | No shadcn; vanilla JS + single HTML file |
| Component library | none | Vanilla JS + CSS custom properties only |

## Phase 1 Stub Layout Contract

The Phase 1 `index.html` stub must implement exactly this layout to serve as a working target for Phase 4 frontend development.

### Page Structure

```
<body>                          # bg: --color-bg
  <main>                        # max-width: 640px, centered, padding: --space-lg
    <h1>                        # Display typography, --color-text
    <section id="connect">      # Surface card, padding: --space-lg, border-radius: 8px
      <button id="btn-connect"> # Accent background, body text, --space-sm / --space-md padding
    <section id="event-log">    # Surface card, padding: --space-lg, border-radius: 8px
      <div.event-item>          # Each SSE event row
```

### Event Log Item Structure

Each SSE named event received on the `EventSource` renders one row:

```
[step indicator dot]  [event name]  [message]       [timestamp]
●                     videos_fetched  Fetched 48...  0.8s
```

- Step indicator dot: 10px circle; color `--color-accent` for in-progress, `--color-success` for terminal `course_assembled`, `--color-destructive` for `error`
- Event name: 14px, `--color-text-muted`, monospace (`font-family: monospace`)
- Message: 16px, `--color-text`
- Timestamp: 14px, `--color-text-muted`, right-aligned

### Progress Bar

A horizontal progress bar below the Connect button shows `step / total` (values from SSE payload):

- Track: `--color-surface-raised`, height: 4px, border-radius: 2px
- Fill: `--color-accent`, width: `calc((step / total) * 100%)`, transition: `width 300ms ease`
- Resets to 0 when a new connection is opened

### Connection States

| State | Button label | Button behavior |
|-------|-------------|-----------------|
| Idle | "Connect to SSE" | Enabled, accent background |
| Connecting | "Connecting..." | Disabled, `opacity: 0.6` |
| Streaming | "Streaming..." | Disabled, `opacity: 0.6` |
| Complete (success) | "Connect to SSE" | Re-enabled; progress bar stays at 100% in `--color-success` |
| Complete (error) | "Connect to SSE" | Re-enabled; progress bar accent becomes `--color-destructive` |

## Implementation Notes for Executor

1. Declare all CSS custom properties in a single `:root {}` block at the top of the `<style>` tag in `index.html`.
2. Use `document.addEventListener('DOMContentLoaded', ...)` — no module bundler, no import/export in Phase 1.
3. `EventSource` connects to `GET /api/course-stream`. Add listeners for each named event: `query_generated`, `videos_fetched`, `scored`, `transcripts_fetched`, `course_assembled`, `error`.
4. On `course_assembled` or `error`: call `eventSource.close()` immediately to prevent auto-reconnect (INFR-05).
5. Heartbeat is a comment line (`: heartbeat\n\n`) — `EventSource` ignores it silently; no listener needed.
6. All JavaScript is in a single `<script>` tag at the bottom of `<body>`. No external JS files in Phase 1.
7. `meta name="color-scheme" content="dark"` in `<head>` to opt the browser chrome into dark mode.
8. `meta name="viewport" content="width=device-width, initial-scale=1"` — required for 375px mobile responsiveness (FRNT-10).

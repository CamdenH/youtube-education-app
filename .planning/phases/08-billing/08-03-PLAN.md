```javascript
// STEP 8c: Usage gate preflight check (D-08, D-09)
// Frontend calls GET /api/usage-check with fetch() before opening EventSource.
// Native EventSource cannot read HTTP status codes — this route exposes the 429.
app.get('/api/usage-check', requireUser, async (req, res) => {
  let usageResult;
  try {
    usageResult = await checkUsage(req.userId);
  } catch (err) {
    console.error('[usage-check] checkUsage failed:', err.message);
    return res.status(500).json({ error: 'Usage check failed.' });
  }
  if (!usageResult.allowed) {
    const limitWord = usageResult.limit === 1 ? 'course' : 'courses';
    return res.status(429).json({
      error: 'usage_limit_reached',
      message: `You've used your ${usageResult.limit} free ${limitWord} this month. Upgrade to Early Access for 20/month.`,
      upgradeUrl: process.env.CLERK_ACCOUNT_PORTAL_URL,
    });
  }
  return res.status(200).json({ allowed: true });
});
```

IMPORTANT: Register this route AFTER `app.use(express.json())` (STEP 6) so req.body parsing works. Register it BEFORE `app.get('/api/transcript/:videoId', ...)` (STEP 9).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → GET /api/usage-check | requireUser enforces authentication before checkUsage runs |
| client → GET /api/course-stream | requireUser + gate check both server-side; 429 fires before any SSE data |
| server.js → process.env.CLERK_ACCOUNT_PORTAL_URL | Env var is server-controlled — never user input; safe to embed in JSON response |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-8-03-01 | Elevation of Privilege | GET /api/course-stream gate | mitigate | Gate calls `db.checkUsage(req.userId)` where req.userId is set by requireUser from Clerk JWT. User cannot inject a different clerkId. |
| T-8-03-02 | Spoofing | GET /api/usage-check auth | mitigate | Route uses requireUser middleware — returns 401 if Clerk JWT is missing or invalid. |
| T-8-03-03 | Tampering | upgradeUrl in 429 response | accept | upgradeUrl comes from `process.env.CLERK_ACCOUNT_PORTAL_URL` (server-controlled). Not from request input. No open redirect risk from user-supplied data. |
| T-8-03-04 | Denial of Service | Gate check Supabase query | accept | One extra Supabase read per course-stream request. Acceptable at launch scale; no additional mitigation needed. |
| T-8-03-05 | Tampering | Counter increment race | mitigate | incrementGenerationCount uses Postgres RPC (atomic UPDATE). Two concurrent requests increment to 2, not 1. Atomic execution in db.js Plan 02 prevents TOCTOU. |
</threat_model>

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight label spacing |
| sm | 8px | Input padding, compact element spacing |
| md | 16px | Default element padding, form field spacing |
| lg | 24px | Section padding, card interior padding |
| xl | 32px | Layout gaps, form bottom margin |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:

| Element | Value | Justification |
|---------|-------|---------------|
| Header bar height | 48px | Aligns with touch target minimum and existing app proportions (maps to 2xl token) |
| Clerk user-button | 32px diameter minimum | Clerk default — do not override (maps to xl token) |
| Focus ring | `outline: 2px solid var(--color-accent); outline-offset: 2px` | Carry forward from existing `index.html` pattern — 2px is a border, not a spacing token |
| `#btn-generate` in `index.html` (existing) | `padding: 10px 20px` | Pre-existing value in `index.html` — 10px does NOT conform to the spacing scale. Do NOT carry this forward. New buttons in `landing.html` and `onboarding.html` MUST use `padding: 8px 16px` (sm/md tokens). `#btn-generate` is not modified in Phase 6; it will be corrected in a future cleanup pass. |

Source: CSS custom properties in `index.html` `:root` — `--space-xs` through `--space-3xl`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.15 |

All four sizes apply to the three new HTML files created in Phase 6:
- `landing.html`: Display (hero headline), Heading (section headers), Body (feature copy), Label (nav links, legal micro-copy)
- `onboarding.html`: Heading (step headers), Body (explanatory copy), Label (metadata)
- `index.html` header bar: Label (user name, sign-out link)

Source: CSS custom properties in `index.html` `:root` — `--font-size-*`, `--font-weight-*`, `--line-height-*`. No changes to these tokens.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f0f0f` (`--color-bg`) | Page background of all three HTML files |
| Secondary (30%) | `#1a1a1a` (`--color-surface`) + `#242424` (`--color-surface-raised`) | Cards, header bar background, onboarding step panels, input fields |
| Accent (10%) | `#3b82f6` (`--color-accent`) | CTA buttons only (landing hero CTA, onboarding CTA, generate button), focus rings, active link underline |
| Destructive | `#ef4444` (`--color-destructive`) | Sign-out confirmation only (if a confirmation dialog is ever added — not required in Phase 6) |

Accent reserved for:
1. Primary CTA button background on `landing.html` ("Sign up free")
2. Primary CTA button background on `onboarding.html` ("Start learning")
3. Focus rings on all interactive elements (`outline: 2px solid var(--color-accent)`)
4. Input border on focus state in `index.html`

Accent is NOT used for: decorative elements, icons, nav links at rest, secondary/ghost buttons.

Source: CSS custom properties in `index.html` `:root` — `--color-bg`, `--color-surface`, `--color-surface-raised`, `--color-accent`, `--color-destructive`. All carry forward unchanged. `landing.html` and `onboarding.html` must declare the same `:root` block.

## Component Inventory

New UI surfaces introduced in Phase 6:

### Header Bar (`index.html`)
- Fixed to top of page, full viewport width
- Height: 48px
- Background: `--color-surface` (`#1a1a1a`)
- Bottom border: `1px solid --color-surface-raised`
- Layout: flexbox, `align-items: center`, `justify-content: space-between`
- Left slot: app name "YouTube Learning Curator" at `--font-size-label`, `--color-text-muted`
- Right slot: Clerk `<clerk-user-button>` component (rendered by Clerk JS CDN)
- Main content area: add `padding-top: 48px` to `main` to prevent header overlap

### Landing Page (`landing.html`) — new file
- Page structure: header (nav) → hero section → features section → CTA section
- Hero headline: Display size (28px, semibold), `--color-text`
- Hero subheading: Body size (16px, regular), `--color-text-muted`
- Primary CTA button: "Sign up free" — `--color-accent` background, white text, 6px border-radius, **padding: 8px 16px** (sm/md tokens)
- Secondary link: "Go to app" — plain text link, `--color-accent` color, no button chrome
- Feature items: 3 items max, Heading size label + Body size description
- Footer: minimal — label size, `--color-text-muted`
- Max content width: 640px centered (matches existing `main` constraint)

### Onboarding Page (`onboarding.html`) — new file
- Single scrolling page, no steps/wizard
- Four content sections (from D-03):
  1. How course generation works — Heading + Body
  2. Skill level options explained — Heading + Label-sized list items
  3. Static example of a finished course — HTML mockup in a `--color-surface` card, max 3 video items shown, `--color-surface-raised` card background
  4. CTA to `/app` — "Start learning" button, accent background, **padding: 8px 16px** (sm/md tokens)
- Max content width: 640px centered
- No server interaction — fully static

### Clerk User Button (app header)
- Rendered by `@clerk/clerk-js` CDN script: `Clerk.mountUserButton(element)`
- Do NOT override Clerk component styles — Clerk handles its own dark mode via `appearance` prop if needed
- Appearance prop: `{ baseTheme: dark }` if Clerk JS v6 supports it; otherwise accept Clerk default
- Container element: `<div id="clerk-user-button"></div>` in header right slot

## Interaction States

All interactive elements must implement these states consistently across all three HTML files:

| State | Visual treatment |
|-------|-----------------|
| Default | As declared in color contract |
| Hover (button) | `opacity: 0.9` on accent background buttons |
| Focus | `outline: 2px solid var(--color-accent); outline-offset: 2px` |
| Disabled | `opacity: 0.5; cursor: not-allowed` |
| Loading | Not applicable in Phase 6 (no async UI operations) |
